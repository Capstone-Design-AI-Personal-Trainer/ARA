# Rehab Recommendation Process

ARA의 재활 추천은 병명을 확정하는 기능이 아니라, 사용자의 현재 상태를 바탕으로 안전한 운동 방향을 제안하는 흐름으로 설계한다.

## 핵심 흐름

1. 상태 입력
   - 불편한 부위
   - 통증 정도
   - 통증 기간
   - 아픈 동작
   - 저림, 붓기, 열감, 힘 빠짐 여부
   - 최근 부상 여부
   - 운동 가능 시간과 평소 운동 수준

2. 위험 신호 확인
   - 통증이 매우 심함
   - 관절이 붓거나 뜨거움
   - 외상 후 움직이기 어려움
   - 저림, 감각 이상, 힘 빠짐
   - 열이 나거나 통증이 계속 악화됨

   위험 신호가 있으면 운동 추천보다 병원 또는 전문가 상담을 우선 안내한다.

3. 현재 상태 요약
   - 사용자의 입력을 짧게 정리한다.
   - 예: "어깨 통증이 있고 팔을 들어 올릴 때 불편하며, 통증은 보통 수준입니다."

4. 가능한 원인 후보 제시
   - 병명을 확정하지 않는다.
   - "가능성", "의심", "참고 분석" 표현을 사용한다.
   - 예: 어깨 충돌 증후군 가능성, 회전근개 약화 가능성, 자세 불균형 가능성

5. 운동 가능 수준 판단
   - 통증 0~2: 가벼운 강화 운동 가능
   - 통증 3~5: 스트레칭, 가동성, 저강도 운동 위주
   - 통증 6 이상: 운동보다 휴식 및 전문가 상담 권고

6. 오늘의 재활 목표 설정
   - 통증을 키우지 않기
   - 움직임 범위 확인하기
   - 약한 근육을 가볍게 활성화하기

7. 맞춤 루틴 추천
   - 3~5개 운동만 추천한다.
   - 각 운동에는 추천 이유, 세트/횟수, 주의사항, 중단 기준을 함께 제공한다.
   - LLM이 운동을 새로 만들어내기보다 ARA가 보유한 운동 목록에서 고르게 한다.

8. 운동 중 체크
   - pose 인식으로 자세 정확도, 속도, 좌우 균형, 보상 동작을 확인한다.
   - 통증 증가, 저림, 어지러움, 날카로운 통증이 있으면 중단을 안내한다.

9. 운동 후 조정
   - 운동 후 통증 변화를 확인한다.
   - 통증 감소: 같은 루틴 유지 또는 약간 강화
   - 변화 없음: 강도 유지, 동작 수정
   - 통증 증가: 강도 낮춤, 일부 운동 제외

## LLM 사용 원칙

- 진단 확정이 아니라 추천 보조로 사용한다.
- 위험 신호 판단은 LLM 전에 규칙 기반으로 먼저 처리한다.
- LLM 출력은 구조화된 JSON 형태로 받는다.
- 추천 운동은 검증된 운동 DB에서 선택하게 한다.
- 항상 "의료진의 진단을 대체하지 않는다"는 안내를 포함한다.

## 요약

```text
상태 입력
→ 위험 신호 확인
→ 현재 상태 요약
→ 가능한 원인 후보
→ 운동 가능 수준 판단
→ 오늘의 재활 목표
→ 맞춤 운동 루틴 추천
→ 운동 중 자세 체크
→ 운동 후 결과 정리
→ 다음 루틴 조정
```
